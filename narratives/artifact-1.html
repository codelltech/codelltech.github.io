Artifact Description

This artifact is a mobile Inventory Manager application originally developed in CS360. The application allows users to authenticate, manage inventory records, and perform CRUD operations through a mobile interface. For this enhancement, the application was refactored and expanded, including improvements to authentication, structure, and security, as well as a full recreation in iOS using SwiftUI and SwiftData.

Justification for Inclusion

I selected this artifact because it demonstrates my ability to evaluate and improve an existing application using software engineering best practices. The project highlights key competencies such as application architecture, authentication design, maintainability, and cross-platform development. These skills directly support my goal of transitioning into a software engineering role.

Enhancements Made

The most significant improvement was restructuring the authentication system by separating login and registration logic into distinct methods. This improved clarity, maintainability, and scalability. Security was strengthened by implementing password hashing with unique salts, ensuring that user credentials are no longer stored in plaintext. Additional improvements included introducing a shared BaseActivity class to reduce duplicated code, removing unused parameters, and reorganizing logic into smaller, reusable components. The application was also recreated in SwiftUI, demonstrating the ability to apply consistent design principles across platforms.

Reflection on the Enhancement Process

This enhancement reinforced the importance of separation of concerns and modular design. Initially, combining login and registration into a single method seemed efficient, but refactoring it into separate methods greatly improved readability and maintainability. One challenge I faced was making structural changes without breaking existing functionality, which required careful testing and incremental updates.

Rebuilding the application in SwiftUI introduced additional challenges, particularly transitioning from Android’s Activity-based architecture to a declarative UI model. This process strengthened my understanding of cross-platform development and reinforced core design principles that apply across different technologies.

This artifact demonstrates my ability to design and evaluate computing solutions using software engineering principles, implement secure coding practices, and refactor code for maintainability. It also reflects growth in applying modern tools and techniques across multiple platforms while effectively communicating technical improvements.
