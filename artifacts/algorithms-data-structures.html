Artifact Overview

This artifact is a course planning application originally developed in C++ for CS300: Data Structures and Algorithms. The program was designed to read course data from a file, store it using a hash table, and allow users to search and display course information, including prerequisites. The original implementation focused on building core data structures manually, especially a custom hash table for efficient storage and retrieval.

Purpose of the Enhancement

The purpose of this enhancement was to preserve the original algorithmic concepts while improving readability, maintainability, and scalability. In the enhanced version, I transitioned the application to Python while keeping the same underlying logic. This allowed me to demonstrate the same core data structures and algorithms in a higher-level language while improving the program’s overall design and organization.

Enhancements Implemented

  • Recreated the application in Python while preserving the original logic
  • Used a Python dictionary to represent the hash table for efficient lookups
  • Added sorting of course data for cleaner and more user-friendly display
  • Restructured the project into a more modular, object-oriented design
  • Introduced classes such as Course and CourseCatalog
  • Split the program into multiple files for improved organization
  • Added better error handling to improve reliability and usability

These enhancements improved code readability and maintainability while also demonstrating my ability to refactor an existing project into a more scalable and professionally structured solution.

Skills Demonstrated

  • Efficient data storage and retrieval
  • Algorithmic problem solving
  • Object-oriented design and modular programming
  • Refactoring and code maintainability
  • File processing and error handling

Connection to Course Outcomes

This artifact demonstrates my ability to design and evaluate data structures for efficient storage and retrieval, apply algorithmic thinking to solve computing problems, and refactor existing code to improve maintainability and performance. The enhanced version also reflects a stronger understanding of how design decisions influence readability, scalability, and long-term usability.

Reflection

Through this enhancement process, I learned how to translate a lower-level implementation into a higher-level language while preserving the same core algorithmic concepts. One of the biggest decisions I faced was whether to closely replicate the original C++ implementation or adopt a more Pythonic approach. I ultimately chose to use Python’s built-in dictionary to represent the hash table because it improved readability and efficiency while still demonstrating the same underlying principles.

Another challenge was restructuring the application into a more object-oriented design. Separating responsibilities between classes such as Course and CourseCatalog required careful planning, but it greatly improved the organization and maintainability of the project. Overall, this process strengthened my understanding of data structures, algorithms, and software design, while also improving my ability to refactor and enhance existing code.
