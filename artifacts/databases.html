Artifact Overview

This artifact is an enhanced version of my CS340 Project Two dashboard application. The original project consisted of a Python-based CRUD module connected to a MongoDB database and a Dash/Plotly dashboard used to visualize animal shelter data. For this enhancement, I expanded the functionality of both the database layer and the dashboard to improve performance, reliability, and overall design. The updated version now includes input validation, indexed queries, aggregation-based reporting, improved error handling, and the ability to run on a locally hosted MongoDB instance.

Purpose of the Enhancement

The purpose of this enhancement was to strengthen the application’s database functionality while making the project more representative of a real-world data-driven system. The original artifact provided a strong foundation, but there were clear opportunities to improve efficiency, reliability, and usability. These enhancements allowed me to refine both the backend query logic and the frontend dashboard experience while demonstrating more advanced database concepts.

Enhancements Implemented

  • Added input validation to improve data integrity and reduce invalid entries
  • Implemented indexed queries to improve database performance
  • Added aggregation-based reporting to move processing closer to the database
  • Improved error handling for stronger reliability and debugging support
  • Refined filtering logic to better align backend queries with frontend components
  • Transitioned the project to run on a locally hosted MongoDB instance
  • Improved the dashboard experience and overall usability of the application

Together, these enhancements improved the scalability, maintainability, and practical value of the project while demonstrating stronger database design and full-stack development skills.

Skills Demonstrated

  • MongoDB database design and integration
  • CRUD operations and input validation
  • Query optimization through indexing
  • Aggregation pipelines and database-side processing
  • Error handling and debugging
  • Interactive dashboard development with Dash and Plotly
  • Full-stack thinking in data-driven applications

Connection to Course Outcomes

This artifact demonstrates my ability to design and improve computing solutions that rely on well-structured data management. It shows growth in database functionality, efficiency, and reliability through the use of validation, indexing, aggregation pipelines, and improved error handling. The project also reflects my ability to connect backend database logic with frontend data visualization in a way that supports a more effective user experience.

By enhancing both the technical design and practical usability of the application, this artifact supports course outcomes related to database development, maintainable system design, and the implementation of scalable data-driven solutions.

Reflection

This enhancement process gave me a much deeper understanding of how database design decisions affect both application performance and usability. One of the most important lessons I learned was the value of moving data processing closer to the database through aggregation rather than relying entirely on application-side logic. I also gained a stronger appreciation for the importance of indexing, especially when working with larger datasets and more complex filtering operations.

One of the biggest challenges I encountered was troubleshooting environment and dependency issues while transitioning from a remote academic environment to a local MongoDB instance. Resolving connection errors and missing dependencies required careful debugging and reinforced the importance of environment management in software development. Identifying and correcting issues in the filtering logic also helped me better understand how backend queries and frontend components must align to produce accurate and useful results.

Overall, this artifact represents meaningful growth in my ability to build robust, data-driven applications. It strengthened my skills in debugging, optimization, database design, and full-stack development, while also demonstrating my ability to improve an existing system in ways that make it more reliable, efficient, and professionally relevant.
