Artifact Overview

This artifact is a mobile inventory management application originally developed in Android Studio using Java and SQLite. The application allows users to create an account, log in, and manage inventory items through create, read, update, and delete (CRUD) functionality. For this enhancement, the project was expanded and recreated as a native iOS application using Xcode, SwiftUI, and SwiftData, demonstrating cross-platform development and adaptability to different technology stacks.

Purpose of the Enhancement

The goal of this enhancement was to improve the overall design, security, and maintainability of the application while transforming it into a more polished, professional artifact. The original version was functional but contained opportunities for improvement in structure, validation, and security. These enhancements focused on applying software engineering best practices and strengthening the application’s architecture.

Enhancements Implemented

  • Refactored authentication by separating loginUser() and registerUser() methods
  • Implemented password hashing with unique salts to improve security
  • Added input validation for email formatting and numeric inventory values
  • Introduced a shared BaseActivity class to reduce duplicated UI logic
  • Removed unused parameters and redundant methods to simplify the codebase
  • Reorganized logic into smaller, reusable methods for better maintainability
  • Recreated the application in SwiftUI and SwiftData for iOS

These enhancements improved the clarity, scalability, and security of the application while demonstrating the ability to evaluate and refactor an existing codebase into a more maintainable solution.

Skills Demonstrated

  • Software refactoring and modular design
  • Secure authentication and data handling
  • Object-oriented programming principles
  • Cross-platform development (Android and iOS)
  • Code maintainability and readability improvements
  • Application architecture and design patterns

Connection to Course Outcomes

This artifact demonstrates the ability to design and evaluate computing solutions using software engineering principles. The restructuring of authentication logic, improved modular design, and removal of unnecessary components reflect a strong understanding of maintainable software design. Additionally, the implementation of password hashing and input validation demonstrates the application of secure coding practices.

The cross-platform implementation further highlights the ability to apply consistent design principles across different development environments, reinforcing competency in modern tools and techniques.

Reflection

Enhancing this artifact reinforced the importance of separation of concerns and modular design. Initially, combining login and registration into a single method seemed efficient, but refactoring into separate methods significantly improved clarity, scalability, and ease of testing.

Rebuilding the application in SwiftUI introduced new challenges, including transitioning from Android’s Activity-based structure to a declarative UI framework and moving from direct SQL queries to object-based persistence with SwiftData. These challenges strengthened my ability to adapt across platforms while maintaining consistent design principles.

Overall, this enhancement deepened my understanding of secure coding, maintainability, and cross-platform development, while demonstrating my ability to evaluate existing code and implement meaningful improvements.
